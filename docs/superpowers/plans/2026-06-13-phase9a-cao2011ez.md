# Phase 9a — Cao2011EZ Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring the Cao2011EZ (Epstein–Zin) model green end-to-end through the public API on **both** C++ Jacobian backends (adept autodiff + SymPy), matching a freshly captured old-toolbox golden.

**Architecture:** Four gaps, all confirmed by the §1.2 probe in the spec: (1) multiple inline reductions per statement, (2) embedded named-interp call — both fixed by generalizing `parseModel`'s hoisting (backend-neutral); (3) `qp_GRID(n)` per-shock indexing — fixed by a one-line parser `_GRID` normalization (both backends already handle `qp(const)`); (4) `var_tensor` — a MATLAB-side feature (ndgrid tensors feeding `inbound` bounds + `initial` interp), invisible to the C++ residual, so no codegen/SymPy tensor work. Tensors are **not** popped to C++ and there is **no** `IterRslt.var_tensor` field (confirmed from the old generated `iter_Cao2011EZ.m`).

**Tech Stack:** MATLAB R2025b (`matlab -batch`), `matlab.unittest`; the new `+gdsge` package (`+parser`, `+ir`, `+codegen`); C MEX (MSVC); SymPy via `pyenv` (uv venv) for the alternative backend.

**Spec:** `docs/superpowers/specs/2026-06-13-phase9a-cao2011ez-var-tensor-design.md` (read §1.2, §5, §6 first).

**Authoritative parity references** (old generated, in `base_package/gdsge/tests/Cao2011EZ/`):
`iter_Cao2011EZ.m` (rng seed `0729`; tensor→bounds/initial wiring lines 50-116; no `var_tensor` in `IterRslt`), `simulate_Cao2011EZ.m` (per-period stripped tensor recompute lines 117-159).

…and append the helper (uses the existing `parseNamedInterpCall` machinery by reading the call span directly):

```matlab
function [icStmts, rest, counter] = hoistEmbeddedInterp(rest, counter)
icStmts = {};
while true
    pos = findEmbeddedInterp(rest);   % index of a name starting an embedded call, or 0
    if pos == 0; break; end
    % span: name ' ( ... )
    openPar = pos + 2;
    closePar = matchParen(rest, openPar);
    interpName = rest(pos).text;
    % parse the args between ( and )
    argToks = [rest(openPar+1:closePar-1), eofTok(rest(closePar))];
    args = {}; q = 1;
    if ~(strcmp(argToks(1).type,'eof'))
        while true
            [a, q] = gdsge.parser.parseExpr(argToks, q);
            args{end+1} = a; %#ok<AGROW>
            if strcmp(argToks(q).type,'punct') && strcmp(argToks(q).text,','); q=q+1;
            elseif strcmp(argToks(q).type,'eof'); break;
            else; error('gdsge:parser:badInterpCall','bad interp args (line %d)', rest(pos).line); end
        end
    end
    counter = counter + 1;
    genName = sprintf('GDSGE_INTERP_%d', counter);
    icStmts{end+1} = struct('type','interpCall','targets',{{genName}}, ...
        'primed',1,'args',{args},'interpRef',interpName); %#ok<AGROW>
    % replace span [pos .. closePar] with a single primed token genName'
    nameTok = rest(pos); nameTok.type='name'; nameTok.text=genName;
    primeTok = rest(pos); primeTok.type='prime'; primeTok.text='''';
    rest = [rest(1:pos-1), nameTok, primeTok, rest(closePar+1:end)];
end
end

function pos = findEmbeddedInterp(rest)
% A name,prime,'(' triple that is NOT the entire RHS (i.e. something other
% than eof follows the matching close paren, OR something precedes the name).
pos = 0;
for i = 1:numel(rest)-2
    if strcmp(rest(i).type,'name') && strcmp(rest(i+1).type,'prime') ...
            && strcmp(rest(i+2).type,'punct') && strcmp(rest(i+2).text,'(') ...
            && ~strcmp(rest(i).text,'GDSGE_INTERP_VEC')
        cp = matchParen(rest, i+2);
        isWholeRhs = (i == 1) && strcmp(rest(cp+1).type,'eof');
        if ~isWholeRhs; pos = i; return; end
    end
end
end

function cp = matchParen(rest, openIdx)
depth = 0;
for i = openIdx:numel(rest)
    if strcmp(rest(i).type,'punct') && strcmp(rest(i).text,'('); depth=depth+1;
    elseif strcmp(rest(i).type,'punct') && strcmp(rest(i).text,')')
        depth=depth-1; if depth==0; cp=i; return; end
    end
end
error('gdsge:parser:badInterpCall','unmatched "(" in interp call');
end
```

After the pre-pass, `icStmts` are prepended to whatever `stmtList` the rest of `parseStatement` produces — change every `stmtList = {…}` / `stmtList = [...]` return in `parseStatement` to `stmtList = [icStmts, …]`. (Simplest: capture the existing return into a local `tail` and do `stmtList = [icStmts, tail];` once at the end. If a single function-wide rewrite is cleaner, wrap the existing body in a subfunction `tail = parseStatementCore(target, primed, rest, …)` and return `[icStmts, tail]`.)

> `eofTok` already exists as a local in `parseModel.m`. `analyzeModel` treats interpCall targets as `primeable` (analyzeModel.m:70-71), so the host's `GDSGE_INTERP_1'` resolves.
