# Phase 9b — CaoNie2016 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring the CaoNie2016 model green end-to-end through the public API on **both** C++ Jacobian backends (adept autodiff + SymPy), matching a freshly captured old-toolbox golden — by adding **conditional model regions** (`model(X>0)`/`model(X==0)`), **`if/else/end` conditional equations**, and **`var_others`** wiring.

**Architecture:** The IR `model` section evolves to a **regions list** (`irVersion 1.1.0`), each region `{condition, statements, equations}`, with equations a **tagged list** (`plain | conditional`). A `regionView(ir,k)` adapter synthesizes the flat `ir.model.statements/equations` the per-body emitters already read, so only the **drivers** (autodiff `emitModel`, sympy `emitTask`, `analyzeModel`) loop regions — the per-body emitters and their unit tests are untouched. Both backends emit each region's body inside a `MODEL_CONDITION` guard (the multi-`MODEL_NUMBER` machinery the templates already support) and each if/else equation as a per-slot runtime branch. A shared `lowerCondition` helper translates the opaque gmod guard to C++. The migration to the 1-region schema lands first (full suite green), then CaoNie2016's new features layer on top.

**Tech Stack:** MATLAB R2025b (`matlab -batch`), `matlab.unittest`; the `+gdsge` package (`+parser`, `+ir`, `+codegen`); C MEX (MSVC); SymPy via `pyenv` (uv venv).

**Spec:** `docs/superpowers/specs/2026-06-14-phase9b-caonie2016-design.md` (read §1.2 probe, §5 detail, §6 golden first).

**Probe (run, authoritative):** `scratch/probe_caonie2016.m` confirmed: `splitBlocks` fails (`unterminatedBlock`); `parseModel` fails only on `if A==AGood`; **all 20 body statements + all plain equations parse OK**; `var_others`→`otherNames` and empty `var_tensor`→`{}` already handled. The new work is control structure, not expressions.

**Old-toolbox parity:** the old `gdsge_parser` supports `model(<cond>)` (`extract_model_segment`), `if/else` equations (lines 2635–2730), `var_others` (lines 306, 1560). CaoNie2016 has ≤1 literal `GNDSGE_EXPECT` per line (`Eqp` is a hoisted variable), so the Phase-9a two-EXPECT blocker does NOT apply — the old toolbox can regenerate the golden.

```matlab
function [modelCode, callFMinCode, numEq] = emitModel(ir)
import gdsge.codegen.cxx.fillTemplate
import gdsge.codegen.cxx.readTemplate
regions = ir.model.regions;
modelParts = cell(1, numel(regions));
callParts  = cell(1, numel(regions));
numEqs = zeros(1, numel(regions));
suffixNames = [cellfun(@(s) s.name, ir.interp, 'UniformOutput', false), ...
    {'GDSGE_INTERP_VEC', 'GDSGE_INTERP_RSLT'}];
for k = 1:numel(regions)
    v = gdsge.ir.regionView(ir, k);
    body = gdsge.codegen.cxx.emitModelBody(v);
    for i = 1:numel(suffixNames)
        body = regexprep(body, ...
            ['((?<=^)|(?<=\W)|(?<=_))' suffixNames{i} '(?=\W)'], [suffixNames{i} '_adouble']);
    end
    [eqCode, numEqs(k)] = gdsge.codegen.cxx.emitEquations(v);
    lam = fillTemplate(readTemplate('model.tpl.cpp'), { ...
        'ARG_CODE',               gdsge.codegen.cxx.emitArgUnpack(ir); ...
        'DECLARE_CODE',           gdsge.codegen.cxx.emitDeclare(v); ...
        'MODEL_CODE',             body; ...
        'HEADER_AUX_ASSIGN_CODE', gdsge.codegen.cxx.emitAux(ir); ...
        'EQUATION_CODE',          eqCode});
    lamDouble = regexprep(lam, '((?<=^)|(?<=\W)|(?<=_))adouble(?=\W)', 'double');
    evalCode = fillTemplate(readTemplate('model_eval.tpl.cpp'), { ...
        'PRE_JAC_CODE',  ir.hooks.preJacCode; 'POST_JAC_CODE', ir.hooks.postJacCode});
    part = [lam newline lamDouble newline evalCode];
    part = fillTemplate(part, {'NUM_EQUATIONS', num2str(numEqs(k))});
    assert(contains(part, 'MODEL_NUMBER'), 'MODEL_NUMBER not found — template mismatch');
    part = strrep(part, 'MODEL_NUMBER', sprintf('MODEL_%d', k));
    modelParts{k} = part;

    cfm = fillTemplate(readTemplate('call_fmin.tpl.cpp'), { ...
        'MODEL_CONDITION', gdsge.codegen.cxx.lowerCondition(regions{k}.condition, ir); ...
        'NUM_EQUATIONS',   num2str(numEqs(k))});
    cfm = strrep(cfm, 'MODEL_NUMBER', sprintf('MODEL_%d', k));
    callParts{k} = cfm;
end
```

```matlab
function entries = parseEquationRegion(eqText, eqOpen)

% else/end` (no ';') are control. One level of nesting only.
lines = regexp(eqText, '\n', 'split');
entries = {};
i = 1;
while i <= numel(lines)
    t = strtrim(lines{i});
    if isempty(t); i = i + 1; continue; end
    if startsWith(t, 'if ') || strcmp(t, 'if') || ~isempty(regexp(t, '^if\W', 'once'))
        [cond, i] = ifCond(lines, i);
        cases = { struct('cond', cond, 'equations', {{}}) };
        i = i + 1;
        while i <= numel(lines)
            tt = strtrim(lines{i});
            if isempty(tt); i = i + 1; continue; end
            if ~isempty(regexp(tt, '^elseif\W', 'once'))
                c = strtrim(regexprep(tt, '^elseif', ''));
                cases{end+1} = struct('cond', c, 'equations', {{}}); %#ok<AGROW>
                i = i + 1;
            elseif strcmp(tt, 'else')
                cases{end+1} = struct('cond', '', 'equations', {{}}); %#ok<AGROW>
                i = i + 1;
            elseif strcmp(tt, 'end') || strcmp(tt, 'end;')
                i = i + 1; break;
            elseif ~isempty(regexp(tt, '^if\W', 'once'))
                error('gdsge:parser:nestedConditionalEquation', ...
                    'nested "if" inside a conditional equation is not supported (line %d)', eqOpen + i);
            else
                eqEntry = parsePlainEqLine(tt, eqOpen + i);
                cases{end}.equations{end+1} = eqEntry;
                i = i + 1;
            end
        end
        entries{end+1} = struct('kind','conditional','cases', {cases}); %#ok<AGROW>
    else
        entries{end+1} = parsePlainEqLine(t, eqOpen + i); %#ok<AGROW>
        i = i + 1;
    end
end
```

```matlab
function cpp = lowerCondition(cond, ir)
% LOWERCONDITION  Translate a model-region / if-else-equation guard (opaque
%   gmod text on states/shocks/params) to a C++ boolean. Identifiers must be a
%   state, shock, or parameter — each is a C++ local or #define in model scope.
%   '~=' -> '!='; other comparison/logical ops are already valid C++. Empty
%   condition -> always true.
cond = strtrim(cond);
if isempty(cond); cpp = '1'; return; end
known = [reshape(ir.states.names, 1, []), reshape(ir.shocks.names, 1, [])];
for i = 1:numel(ir.params); known{end+1} = ir.params{i}.name; end %#ok<AGROW>
ids = regexp(cond, '[A-Za-z_]\w*', 'match');
for i = 1:numel(ids)
    if ~ismember(ids{i}, known)
        error('gdsge:codegen:badCondition', ...
            'condition "%s" references unknown identifier "%s" (allow: states, shocks, params)', ...
            cond, ids{i});
    end
end
cpp = strrep(cond, '~=', '!=');
end
```
