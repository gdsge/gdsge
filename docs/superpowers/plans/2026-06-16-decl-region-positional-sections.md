# Declaration-region positional section model — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Classify the `.gmod` declaration region by **position** (statements belong to the preceding declaration's block), feed each block to a distinct generated coding block, warn (don't error) on misplaced known statements, and hard-error on non-contiguous re-opened declarations.

**Architecture:** A new sectioning pass in `parseVarDecls` walks the declaration statements in source order and groups setup statements into ordered sections `{kind, body}`. Eval semantics are unchanged (section bodies concatenated in source order = today's `setupText`). `ir.setup` changes from a single text blob to an ordered list of sections — a **breaking schema change → major version bump to 2.0.0**. `emitSetup` replays each section under a labeled comment.

**Tech Stack:** MATLAB R2025b (`+gdsge` package), `matlab.unittest`, schema-driven IR (`gdsge.ir.*`).

**Spec:** `docs/superpowers/specs/2026-06-16-decl-region-positional-sections-design.md`

**Deviation from spec (flagged):** Sections are `{kind, body}` (no `names` field). Declared names already live in `ir.params`/`shocks`/`states`/`variables`; `names` would be redundant and add roundtrip/fixture cost. If the reviewer wants `names` back, add it to the schema struct in Task 5 and populate it in Task 1.

**Section kinds (the `kind` field's allowed values):** `global` (preamble, before the first declaration), `parameters`, `var_shock`, `var_state`, `var_policy`, `var_aux`, `var_interp`, `var_tensor`, `var_output`, `var_others`, `var_policy_init`, `var_aux_init`.

**IMPORTANT — red window:** Tasks 5–8 form one coordinated breaking change (schema flip + every consumer). The full suite is expected to be RED partway through; it returns GREEN at the end of Task 8 and is verified in Task 9. Do not panic at intermediate failures within tasks 5–8.

```matlab
% ---- Pass C: positional sectioning ---------------------------------------
% Group setup statements into ordered {kind, body} sections by the preceding
% declaration. Body holds exactly the statements that go to setupStmts (plain
% value assignments), so the concatenation of bodies in source order equals
% the flat setup text emitSetup replays. Interp-updates and tensor-assigns are
% carried separately (NOT in any body), exactly as setupStmts excludes them.
setupSet = containers.Map('KeyType','char','ValueType','logical');
for k = 1:numel(decls.setupStmts); setupSet(decls.setupStmts{k}) = true; end

sections = {};
curKind = 'global';
curBody = {};
headerOpen = false;
seenKinds = {};
    function flushSection()
        sections{end+1} = struct('kind', curKind, ...
            'body', strjoin(curBody, sprintf('\n'))); %#ok<AGROW>
        curBody = {};
    end
for i = 1:numel(stmts)
    st = stmts{i};
    kw = regexp(st, '^([A-Za-z_]\w*)', 'tokens', 'once');
    key = ''; if ~isempty(kw); key = kw{1}; end
    if ismember(key, DECL_KINDS)
        if headerOpen && strcmp(key, curKind)
            % contiguous same-kind declaration line — extend current header
            continue;
        end
        if ismember(key, seenKinds)
            error('gdsge:parser:reopenedDeclBlock', ...
                ['Declaration kind "%s" is re-opened after other statements. ' ...
                 'Make all "%s" declarations contiguous (move them together, ' ...
                 'with no other statements between them).'], key, key);
        end
        flushSection();
        curKind = key; seenKinds{end+1} = key; headerOpen = true; %#ok<AGROW>
    else
        headerOpen = false;
        if isKey(setupSet, st)
            curBody{end+1} = st; %#ok<AGROW>
        end
    end
end
flushSection();
decls.sections = sections;
```

Note: `flushSection` is a nested function, so it shares `sections`/`curKind`/`curBody`. Nested functions require the parent function to **not** use `end`-less form — `parseVarDecls.m` already ends with `end`, so this is fine. If MATLAB complains that the main function mixes nested + local functions, convert `flushSection` to inline code (replace each `flushSection();` call with the two-line body).

function txt = setupText(ir)
% SETUPTEXT  Concatenate ir.setup section bodies into one eval-able string,
%   in source (list) order. Empty bodies are dropped so the result has no
%   blank-line padding. Returns '' for an IR with no setup sections.
txt = '';
if ~isfield(ir, 'setup') || ~iscell(ir.setup); return; end
bodies = {};
for i = 1:numel(ir.setup)
    s = ir.setup{i};
    if isstruct(s) && isfield(s, 'body') && ~isempty(strtrim(s.body))
        bodies{end+1} = strtrim(s.body); %#ok<AGROW>
    end
end
txt = strjoin(bodies, sprintf('\n'));
end
```

    % If the IR declares params/shocks/states but has no setup text the generated file
    % would silently lack all its definitions — a hard-to-diagnose runtime failure.
    hasDecls = ~isempty(ir.params) || ...
               ~isempty(ir.shocks.names) || ...
               ~isempty(ir.states.names);
    if hasDecls
        error('gdsge:codegen:missingSetup', ...
            ['emitSetup: ir.setup is absent or empty but the IR declares params, ' ...
             'shocks, or states. The generated file would lack all variable ' ...
             'definitions. Populate ir.setup with the gmod declaration-region sections.']);
    end
else
    for i = 1:numel(ir.setup)
        sec = ir.setup{i};
        if ~isstruct(sec) || ~isfield(sec, 'body') || isempty(strtrim(sec.body))
            continue;   % declaration-only block (e.g. inbound-only var_policy)
        end
        w.add('%% -- %s', sec.kind);
        w.addRaw(sec.body);
    end
end
```
